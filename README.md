# AI Search Lab

## Overview

AI Search Lab is an interactive web application that demonstrates classical Artificial Intelligence search algorithms through two modules:

### Module A – 8-Puzzle Solver

Implements state-space search algorithms to solve the 8-puzzle problem. Users can shuffle the puzzle, make manual moves, and visualize solution steps generated by search algorithms.

### Module B – Tic-Tac-Toe AI

Implements adversarial search algorithms that allow users to play Tic-Tac-Toe against an AI opponent or observe AI-versus-AI gameplay. Performance metrics are displayed to compare search efficiency.

## Algorithms Implemented

### Module A

#### Breadth-First Search (BFS)

Explores states level-by-level and guarantees the shortest solution path when all actions have equal cost.

#### Dijkstra's Algorithm

Expands states according to cumulative path cost and guarantees an optimal solution.

#### A* Search

Combines path cost and heuristic estimation using:

f(n) = g(n) + h(n)

where:

* g(n) = cost from start state
* h(n) = estimated cost to goal

### Module B

#### Minimax

A complete adversarial search algorithm that evaluates all possible future game states and selects the optimal move.

#### Alpha-Beta Pruning

An optimization of Minimax that eliminates branches that cannot affect the final decision, significantly reducing the number of nodes explored.

## Heuristic Justification

### A* Heuristic

The Manhattan Distance heuristic is used for the 8-puzzle problem.

For each tile:

Manhattan Distance = |current row − goal row| + |current column − goal column|

The heuristic is admissible because it never overestimates the true cost to reach the goal state. Each tile must move at least its Manhattan Distance before reaching its correct position.

Therefore:

h(n) ≤ actual remaining cost

which guarantees optimality when used with A* Search.

## Performance Dashboard

The Tic-Tac-Toe module displays:

* Decision Time (milliseconds)
* Nodes Explored
* Pruning Rate

These metrics allow direct comparison between Minimax and Alpha-Beta Pruning.

---

## Comparative Analysis

Testing showed that Alpha-Beta Pruning significantly reduced the number of nodes explored compared to standard Minimax while producing the same optimal move decisions.

Example Result:

| Algorithm  | Nodes Explored |
| ---------- | -------------: |
| Minimax    |         59,704 |
| Alpha-Beta |          4,089 |

Alpha-Beta Pruning reduced node exploration by approximately 93% while maintaining optimal play quality.
